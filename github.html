Samarbeid på github

Det å skulle samarbeide i repositories som GitHub har vært en utfordring, men har også vært en verdifull læringsprosess. Gruppen ble introdusert for et verktøy som var nytt for majoriteten av medlemmene, og det har tatt lang tid før vi ble komfortable med å bruke dette verktøyet.

Trøblete Terminologi

I starten var det mange fraser og begreper som ikke ga mening for oss, og det tok lang tid før vi skjønte oss på all terminologien, feks forskjellen mellom branching og merge, commit og push osv. Det var også noe trøblete å forstå forskjellen på hvordan GitHub og GitHub desktop fungerte, og hvilket program man skulle benytte seg av når. Dette forlenget definitivt arbeidsprosessen.

Merge Conflicts

Vi opplevde en del merge-conflicts mot slutten. Noe av grunnen til dette var mangelen på klasser i CSS-filen vår, noe vi har tatt lærdom fra. I frykt for å ødelegge for oss selv har vi ved flere anledninger kopiert kode fra en branch, direkte inn i main for å løse merge-conflicts. Dette er lite forenlig med arbeidslivet, og mot slutten av prosjektet har vi jobbet mer med å løse det direkte i github.
